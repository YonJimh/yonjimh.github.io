块格式化上下文BFC

发表于 2019-08-26 | 分类于 Css | 阅读次数：

了解BFC之前先要理解css几个概念，盒子模型，视觉格式化模型

视觉格式化模型

MDN上有讲到 CSS 视觉格式化模型（visual formatting model）是用来处理和在视觉媒体上显示文档时使用的计算规则。该模型是 CSS 的基础概念之一。
视觉格式化模型会根据CSS盒子模型将文档中的元素转换为一个个盒子

什么是 FC（Formatting Contexts）

FC的全称是：Formatting Contexts，是W3C CSS2.1规范中的一个概念。它是页面中的一块渲染区域，并且有一套渲染规则，它决定了其子元素将如何定位，以及和其他元素的关系和相互作用。

BFC

下列方式会创建块格式化上下文：

  • 根元素<html>
  • float的值不为none
  • overflow的值不为visible
  • position的值不为relative和static
  • display的值为table-cell, table-caption, inline-block, flow-root 文档规则
  1. 在一个块格式化上下文中，盒在垂直方向一个接一个地放置，从包含块的顶部开始。
  2. 两个兄弟盒之间的垂直距离由margin属性决定。
  3. 同一个块格式化上下文中的相邻块级盒之间的垂直外边距会合并。
  4. 同一个块格式化上下文中，每个盒的左外边界（left outer edge）挨着包含块的左外边界（对于从右向左的格式化，右外边界挨着）。
  5. BFC的区域不会与float box重叠。
  6. 计算BFC的高度时，浮动元素也参与计算。

详细规则请查看块格式化上下文

作用

  1. 清除浮动：因为BFC可以包含浮动的元素，故把浮动元素的父元素设为BFC容器即可。（ 规则第6点）

    触发了BFC的属性值都可以闭合浮动，但是效果会不一样推荐看一下对overflow与zoom”清除浮动”的一些认识这篇文章。

  2. 防止外边距合并

  3. 不会与浮动重叠 <!– 元素原本的流体特性依然保留了– >

IFC

IFC(Inline Formatting Contexts)直译为”内联格式化上下文”
框会从包含块的顶部开始，一个接一个地水平摆放。

  • IFC的line box（线框）高度由其包含行内元素中最高的实际高度计算而来（不受到竖直方向的padding/margin影响)，不能指定宽高
  • IFC中的line box一般左右都贴紧整个IFC，但是会因为float元素而扰乱。float元素会位于IFC与与line box之间，使得line box宽度缩短。 同个ifc下的多个line box高度会不同。
  • IFC中时不可能有块级元素的，当插入块级元素时（如p中插入div）会产生两个匿名块与div分隔开，即产生两个IFC，每个IFC对外表现为块级元素，与div垂直排列。

GFC

GFC(GridLayout Formatting Contexts)直译为”网格布局格式化上下文”，当为一个元素设置display值为grid的时候，此元素将会获得一个独立的渲染区域

FFC

FFC(Flex Formatting Contexts)直译为”自适应格式化上下文”，display值为flex或者inline-flex的元素将会生成自适应容器（flex container）

CSS2.1中只有BFC和IFC, CSS3中才有GFC和FFC。

外边距合并（margin collapsing）

  1. 兄弟元素外边距；
  2. 父元素与其第一个或最后一个子元素之间
  3. 没有高度的元素自身margin-top和margin-bottom会合并

解决方法：

  1. 给父级元素添加边框属性
  2. 给父级元素添加padding padding-bottom/top
  3. 给父级元素添加overflow：hidden；
  4. 给子级或父级一方添加浮动
  5. 给子级或父级一方添加绝对定位
  6. 给子级或父级一方添加display：inline-block；
