AJAX

发表于 2019-08-31 | 分类于 JavaScript | 阅读次数：

AJAX （Asynchronous Javascript And XML）是什么就不用我废话了，平常开发的时候用到的都是一些封装好的库（比如jq的$.ajax），一开始学习的JavaScript的时候也有了解一下，现在写一篇文章来深入了解一下(〃’▽’〃)。

详细请上MDN查看

XMLHttpRequest 对象

Ajax 技术的核心是 XMLHttpRequest 对象，就是我们常说的XHR对象。

在早期的IE版本（IE7之前）的 XHR 对象是通过 MSXML 库中的一个 ActiveX 对象实现的

上面就是一个简单demo， 我们可以发现一个 XHR 对象的 readyState 属性，该属性表示请求/响应过程的当前活动阶段。这个属性可取的值如下。

  • 0：未初始化。尚未调用 open() 方法。
  • 1：启动。已经调用 open() 方法，但尚未调用 send() 方法。
  • 2：发送。已经调用 send() 方法，但尚未接收到响应。
  • 3：接收。已经接收到部分响应数据。
  • 4：完成。已经接收到全部响应数据，而且已经可以在客户端使用了。

只要 readyState 属性发生变化，都会触发一次 readystatechange 事件，因此想正确的监听每个状态的改变要在调用 open() 之前指定 onreadystatechange 事件处理程序。

常用方法

这里列出几个常用方法（send和open就列出了），其实XHR实例对象的方法也就几个。(〃’▽’〃)，详细请上MDN查看

  • abort() ： 方法将终止该请求。当一个请求被终止，它的 readyState 属性将被置为0
  • getAllResponseHeaders() ： 方法返回所有的响应头，以 CRLF 分割的字符串（回车符（CR）和换行符（LF） \r\n），或者 null 如果没有收到任何响应
  • getResponseHeader(name) : 获取包含指定头文本的字符串。
  • setRequestHeader(header, value) : 设置请求头

事件

onload

响应接收完毕后将触发 load 事件。只要浏览器接收到服务器的响应，不管其状态如何，都会触发 load 事件。而这意味着你必须要检
查 status 属性，才能确定数据是否真的已经可用了。

onprogress

新版本的XMLHttpRequest对象，传送数据的时候，有一个progress事件，用来返回进度信息

它分成上传和下载两种情况。下载的progress事件属于XMLHttpRequest对象，上传的progress事件属于 XMLHttpRequest.upload 对象

  • event.total 是需要传输的总字节
  • event.loaded 是已经传输的字节
  • event.lengthComputable 一个表示进度信息是否可用，如果event.lengthComputable不为真，则event.total等于0

几个比较特殊的属性

这里列出几个我觉得比较特殊的属性。

  • upload 属性

    返回一个 XMLHttpRequestUpload 对象，用来表示上传的进度， 这个对象是不透明的 但是可以绑定事件 比如onprogress

  • timeout 属性 表示请求在等待响应多少毫秒之后就终止

    在给timeout 设置一个数值后，如果在规定的时间内浏览器还没有接收到响应，那么就会触发 timeout 事件，进而会调用 ontimeout 事件处理程序

  • responseType 属性 来改变一个从服务器上返回的响应的数据类型。

    可用的属性值为 空字符串 (默认), “arraybuffer”, “blob”, “document”,”json” 和 “text”. response 属性的值会根据responseType 属性包含实体主体（entity body）, 它可能会是一个 ArrayBuffer, Blob, Document, JSON, string 或者为 NULL(如果请求未完成或失败) （就是说读取 xhr.response 而不是 xhr.responseText） 详细请看这里。下面引用一下例子：

  • withCredentials 属性 可以指定某个跨域请求应该发送凭据

    默认情况下，跨域请求不提供凭据（cookie、HTTP 认证及客户端 SSL 证明等）。通过将 withCredentials 属性设置为 true ，可以指定某个请求应该发送凭据。如果服务器接受带凭据的请求，会用下面的 HTTP 头部来响应（开启CORS）。

    在同一个站点下使用 withCredentials 属性是无效的。 永远不会影响到同源请求。
